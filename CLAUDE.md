1. 本项目后续所有对话与输出（含代码注释、提交信息建议）**统一使用中文**。
2. 优先保证正确性与可维护性，避免过度设计。
3. 任何涉及网络/加密/鉴权的实现需默认安全（本项目节点间 gRPC 使用 mTLS）。
4. 变更尽量小步可运行：每个阶段保证 `go build ./...` 通过。

## 项目概览

这是一个以常驻服务形式运行的 Go 集群组件：

- 单端口 TCP 上运行 gRPC（mTLS），承载三类逻辑通道：
  - 选举通道（主备心跳与选举）
  - 控制通道（Master 广播节点状态/角色变更）
  - 数据通道（端口映射/隧道，转发 TCP/Unix Socket 字节流）
- 角色：Master / Backup / Slave
- 选举：类似 VRRP 的“心跳比较”方案（无多数派，需注意网络分区风险）
- 本地 HTTP API：
  - `POST /elect` 手动指定切换
  - `GET /nodes` 查询节点
  - `GET /subscribe` SSE 订阅状态

## 常用命令（开发时）

- 构建：`go build ./...`
- 运行：`go run ./cmd/cluster -config config/example.json`

## 重要限制

- 本项目不实现多数派/共识协议；网络分区可能导致短暂“双 Master”。
- 节点间通信使用 mTLS，需要准备 CA/证书/私钥。
